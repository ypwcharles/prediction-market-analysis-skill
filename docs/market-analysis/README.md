## Naming Rules

- Use `YYYY-MM-DD-topic.md`.
- Prefer the market or thesis name in the slug, not vague labels such as `analysis-1`.
- If the note is a sector scan, include the sector in the filename.
- If the note consolidates an earlier chat, say so in the first few lines.

## Suggested Note Structure

For consistency, most new notes should include:

1. Analysis date
2. Market link or event identifier
3. Price snapshot date or time range
4. Scope of the note
5. Final verdict
6. Probability and edge discussion
7. Key evidence and rule-scope clarifications

That keeps the archive usable later, when the original discussion context is gone.
