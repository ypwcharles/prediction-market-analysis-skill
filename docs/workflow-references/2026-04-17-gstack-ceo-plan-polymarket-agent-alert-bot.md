## Vision

### 10x Check
This should not be a tracker or another noisy alert bot. It should become a Polymarket chief of staff that wakes up, scans the live board, filters out junk, notices repricing, understands your existing exposure, and only interrupts you when there is a trade you may actually want to make now. The long-term product is not "more data." It is compressed conviction.

### Platonic Ideal
The user does not browse Polymarket to figure out what matters. Telegram already contains a tiny, ruthless morning memo. Most days it says no trade. Some days it says there are exactly one or two things worth attention, with the rule line, evidence card, max entry, overlap with existing positions, and what would invalidate the thesis. The system remembers prior alerts, old rejected ideas, and how the user reacts, so it keeps getting quieter and sharper.

## Scope Decisions

| # | Proposal | Effort | Decision | Reasoning |
|---|----------|--------|----------|-----------|
| 1 | Stateful thesis memory | M | ACCEPTED | Without memory the product is a notifier, not a trading assistant |
| 2 | Repricing / reactivation engine | M | ACCEPTED | A lot of real edge appears when an old thesis becomes tradable again |
| 3 | Telegram feedback and calibration loop | M | ACCEPTED | The bot should learn what is worth interrupting the user for |
| 4 | Automatic position read + overlap judgment | M | ACCEPTED | Recommendations should respect current exposure, not assume an empty book |
| 5 | Evidence + rule-scope explanation card | M | ACCEPTED | Trust is the product; every alert needs a compact investment memo |

## Accepted Scope (added to this plan)

- Agent-first scheduled scanner
- Telegram-first opportunity delivery
- Minimal position monitoring
- Stateful thesis memory
- Repricing / reactivation logic
- Telegram feedback loop for calibration
- Automatic position read and overlap-aware recommendations
- Evidence-source and rule-scope explanation cards on every alert
