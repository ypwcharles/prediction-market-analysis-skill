## Affected Pages/Routes

- Telegram opportunity alerts, verify strict long memo, research digest, monitor action alert, and heartbeat rendering
- Scheduled `scan` job, verify discovery, filtering, judgment routing, persistence, and degraded behavior
- Scheduled `monitor` job, verify trigger evaluation, stale memo handling, and callback-driven state transitions
- Calibration report generation, verify markdown output and SQLite summary consistency

## Key Interactions to Verify

- A new strict candidate is discovered and sent as a long memo with correct labels, links, edge fields, and citations
- A research-only run produces digest output rather than interrupt-style strict messaging
- A user taps each Telegram inline action and the correct SQLite state transition occurs
- A monitor trigger fires after a prior memo and produces the right action-oriented follow-up
- A stale memo stops generating direct action advice until rechecked

## Edge Cases

- Gamma/CLOB fetch partially fails but run still emits degraded status rather than silent success
- Primary evidence sources conflict and candidate is blocked from normal strict promotion
- Telegram send fails and retry/error state is persisted
- Position API lags behind a user-claimed `已下单` action
- Same thesis cluster reprices into a new tradable zone without duplicating old alert state
- Trigger is acknowledged or snoozed and does not immediately re-fire on the next poll

## Critical Paths

- `scan -> strict memo -> archive/sqlite write -> telegram send`
- `scan -> no strict result -> heartbeat with coverage accounting`
- `monitor -> mechanical trigger -> action alert`
- `monitor -> narrative trigger -> LLM recheck -> action alert or downgrade`
- `telegram callback -> feedback state update -> later monitor behavior changes`
